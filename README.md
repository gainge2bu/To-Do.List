# 컨설턴트 일정 관리 시트

> 컨설턴트의 할 일·우선순위·캘린더를 한 화면에서 관리하는 단일 파일 웹앱

🌐 **라이브**: https://gainge2bu.github.io/To-Do.List/

---

## 기능

### 1. 할 일 관리
- **오늘 집중** — 기한 초과 / 오늘 마감 / 이번 주(3일 내) / 예정 항목 4단 분류
- **아이젠하워 매트릭스** — 긴급도 × 중요도 4분면, **드래그&드롭**으로 우선순위 변경
- **자동 긴급 처리** — 마감일이 D-3 이내면 자동으로 긴급 4분면(Q1·Q3)으로 승격
- **캘린더 뷰** — 월간 그리드로 할 일 마감일 시각화
- **전체 보기** — 필터 칩(전체/진행 중/완료/Q1~Q4)으로 정렬

### 2. 구글 캘린더 연동 (OAuth)
- **본인 + 공유 캘린더 자동 로드** — Workspace에 공유된 모든 캘린더 자동 발견
- **일 / 주 / 월 뷰** 토글 + 시간 칸 줌 (20~80px)
- **시간박싱 그리드** — Google Calendar 스타일 주간 뷰, 겹치는 일정 자동 분할
- **CRUD** — 본인 캘린더는 추가·수정·삭제, 공유 받은 캘린더는 읽기 전용(RO)
- **이메일로 캘린더 구독** — 동료 이메일 입력만으로 즉시 추가
- 기본값: **본인 캘린더만 표시**, 동료 캘린더는 수동 토글

### 3. 개인화
- **라이트/다크 모드** 토글 (🌙 ↔ ☀)
- **헤더 커스텀** — 이름·직함·영문 호칭·미션 입력
- **한글 조사 자동 처리** — 받침 유무로 이/가 자동 선택 (`임종은이`, `김조구가`)
- 모든 설정은 브라우저 localStorage에 저장 (탭별로 독립)

## 다른 회사에서 재사용하기

같은 코드를 다른 회사 환경으로 옮기려면 **15~20분**이면 됩니다.

### 1. Google Cloud 프로젝트 생성 (5분)
1. https://console.cloud.google.com → **새 프로젝트** 만들기
2. 프로젝트 이름: 자유 (예: `Company Schedule Sheet`)
3. 조직: 해당 회사 Workspace 도메인 선택

### 2. Google Calendar API 활성화 (1분)
- **API 및 서비스 → 라이브러리** → `Google Calendar API` 검색 → **사용** 클릭

### 3. OAuth 동의 화면 (5분)
1. **API 및 서비스 → OAuth 동의 화면**
2. **User Type: 내부(Internal)** ← 회사 도메인만 접근, 검증 면제
3. 앱 이름·지원 이메일·개발자 이메일 입력

> 외부 도메인 사용자도 쓰게 하려면 **외부(External)** 선택. 단, 검증 안 받으면 총 100명 한도.

### 4. OAuth Client ID 발급 (3분)
1. **사용자 인증 정보 → + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
2. 애플리케이션 유형: **웹 애플리케이션**
3. 승인된 JavaScript 출처: 호스팅 URL 등록 (예: `https://username.github.io`)
4. 생성된 **Client ID 복사**

### 5. 코드 교체 + 배포 (3분)
1. `index.html`에서 한 줄만 교체:
   ```js
   const GCAL_CLIENT_ID = "여기에 새 Client ID 붙여넣기";
   ```
2. GitHub Pages(또는 Vercel·Netlify·회사 정적 호스팅)에 업로드

> 디자인이나 회사 브랜드 컬러를 바꾸려면 `:root` 안의 CSS 변수(`--bg`, `--blue`, `--lime` 등)만 손대면 됩니다.

## 사용 팁

- **오늘 집중 탭이 메인** — 매일 아침 열어서 오늘 할 일부터 처리
- **매트릭스 드래그** — Q4(후순위) → Q1(긴급·중요) 이동만으로 우선순위 재배치
- **마감일 비워둬도 됨** — 입력 없으면 "예정 항목" 섹션에 자동 분류
- **자동 긴급의 의미** — 마감 D-3 이내는 사용자가 "중요"로 설정해도 매트릭스에선 "긴급·중요"로 표시 (저장된 값은 그대로, 표시만 자동 승격)
- **팀 캘린더 새로고침** — 외부에서 일정 추가하고 갱신 안 보이면 우상단 ↻ 클릭

## 기술 스택

- **HTML/CSS/JS 단일 파일** — `index.html` 하나, 빌드 도구 없음

- 프레임워크 없음, 번들러 없음, 종속성 없음
